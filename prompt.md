
请基于以下需求设计**学生信息管理系统**全栈技术方案与代码框架，需满足全部评分项要求，采用现代化技术栈实现企业级实践。

### 一、核心架构要求
1. **前后端分离**  
   - 前端：`Vue3` + `Pinia`状态管理 + `Vue Router`  
   - 后端：`Spring Boot` + `Spring Security` + `Spring Data Redis` + `MySQL`  + `Spring Data JPA` 
   - 通信协议：`RESTful API` + `JWT令牌`

### 二、功能与界面规范（对应需求2.2~2.4）
2. **多层界面逻辑跳转**  
   - 实现路由守卫控制访问层级（如：游客→登录→学生列表→详情页→编辑页）  
   - 动态路由加载（不同角色显示不同菜单）
3. **图片展示**  
   - 学生头像上传/预览（支持裁剪）  
   - 班级相册墙（瀑布流布局 + 懒加载）
4. **CSS样式**  
   - 使用`SCSS`编写模块化样式  
   - 响应式布局（适配移动端）  
   - 主题切换功能（亮色/暗色模式）

### 三、关键技术与评分项实现（对应需求1.4~2.4 & 其他）
5. **用户认证与会话管理**  
   - 基于`Redis`存储会话：`Spring Session + Redis`  
   - 登录流程：用户名密码 → JWT签发 → Redis缓存Token → 前端存储
6. **权限控制系统**  
   - 持久化方案：`MySQL`存储角色-权限映射表  
   - 注解鉴权：`@PreAuthorize("hasRole('ADMIN')")`  
   - 动态权限加载：启动时从DB加载权限树
7. **AOP+Redis+前端展示联动（重点10分项）**  
   - AOP切面：拦截Controller方法记录操作日志  
   - Redis存储：实时操作计数（如：`user:login:count`）  
   - 前端可视化：仪表盘展示实时操作热力图（ECharts）
8. **数据持久层**  
   - `Spring Data JPA` + `Hibernate`  
   - 实体关系：学生↔班级(多对一) ↔ 课程(多对多)
9. **消息队列集成**  
   - `Kafka`应用场景：  
     - Producer：学生注册成功后发送欢迎消息  
     - Consumer：异步处理邮件通知/积分变更  
   - 保证消息可靠性：ACK机制 + 死信队列
10. **Redis深度应用（10分项）**  
    - 缓存策略：  
      - 学生查询接口：`@Cacheable`（TTL=30min）  
      - 热点数据：手动预热Redis  
    - 防击穿：互斥锁（`Redisson`分布式锁）  
    - 数据结构：使用Hash存储学生基本信息
